<CardChapter>

  ## Code Conventions

<CardInfo>

Info:

- Add a semicolon at the end of each executable statement

- wrap line after an operators
  ```javascript 
  document.getElementById("demo").innerHTML =
  "Hello Dolly!";
  ```

</CardInfo>

<CardDanger>

Danger:

- avoid code lines longer than 80 characters

- Hyphens are not allowed in JavaScript. They are reserved for subtractions.

</CardDanger>

</CardChapter>

  ## Syntax

<CardInfo>

Info:

- values
  - Fixed values are called Literals
  - Variable values are called Variables


</CardInfo>

<CardDanger>

Danger:

- An equal sign `=` is used to assign values to variables.
- Numbers are not allowed as the first character in names.

</CardDanger>

<CardChapter>

  ## Placement

<CardInfo>

Info:

- You can place any number of scripts in an HTML document.
- can be placed in the `<body>`, or in the `<head>` section
  - Placing at bottom `<body>` element improves the display speed, because script interpretation slows down the display.


</CardInfo>

- inline

  ```html 
  <!DOCTYPE html>
  <html>
    <body>

      // highlight-start
      <script>
      function foo() {
          console.log("Hello World!");
      }
      </script>
      // highlight-end

    </body>
  </html>
  ```

- outsourced / external file

  ```html 
    <!DOCTYPE html>
    <html>
      <body>

        // highlight-start
        <script src="myScript.js"></script>
        // highlight-end

      </body>
    </html>
  ```

- full url

  ```javascript 
  <script src="https://www.foo/js/myScript.js"></script>
  ```

</CardChapter>

<CardChapter>

  ## Comments

<CardInfo>

Info:

- It is most common to use single line comments.

  Block comments are often used for formal documentation.


</CardInfo>

- line comment

  ```javascript 
  let x, y, z;    // Statement 1
  ```

- block comment
  ```javascript 
  /* 
    block
    comment
  */
  ```
